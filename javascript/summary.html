- 객체 : 프로그램에서 인식할 수 있는 모든 대상 
- instance (사전적의미:사례) : 객체는 new 키워드를 통해 Date객체의 인스턴스를 만들어 변수에 저장하여 사용한다.
ex) let nowDate = new Date()

- property : 객체의 속성
- method : 객체의 동작, 함수와 같은 역할을 하기때문에 ()붙여야 함. ex)nowDate.toLocaleString()

- 매개변수(parameter) : 함수선언 시 데이터를 전달받기 위한 변수
- 인수(argument) : 함수 호출시 매개변수에 넘겨주는 값

- string은 유사배열로써, index와 length를 가진다. 재할당(주소변경)은 가능하나, 변수[0] = "변경값"을 지정할 순 없다.

- let, const로 선언한 전역변수는 window객체의 프로퍼티가 아니다. 
   ex) let foo = 1 => window.foo (X) var선언은 window객체의 프로퍼티이다. 

- let, const는 블록레벨 스코프를 가지며 호이스팅이 발생하지 않는 것처럼 동작한다. 

- const는 선언과 동시에 초기화(할당)이 되어야 함(재할당 금지), 일반적으로 상수는 대문자로 선언하고 언더스코어로 구분(스테이크)
  ex) const TAX_RATE = 0.1

- const로 객체를 선언한 경우는 프로퍼티 키값을 변경할 수 있다. 
  ex) const person = { name: 'lee' } 
     person.name = 'Kim'
     console.log(person) ==> {name:'Kim'}

- 재할당이 필요한 경우 let을 사용하고 최대한 스코프를 좁게 만들어야 한다. 

- 객체의 프로퍼티 어트리뷰트 
   Object.getOwnPropertyDescriptor(person, "name")
  {value: 'Kim', writable: true, enumerable: true, configurable: true} 
  (*enumerable : 열거가능여부, false이면 for in문이나 Object.keys로 열거할 수 없음 / configurable : 재정의 가능여부) 

- getter, setter는 자체적으로 값을 가지지 않고 객체의 키값을 읽거나 저장할 때 사용함

- 프로토타입이란? prototype : 어떤 객체의 상위(부모) 객체, 하위(자식) 객체는 부모의 메서드와 프로퍼티를 상속받는다. 

- nullish 병합연산자 a ?? b ==> a가 null 또는 undefined가 아닌 값을 가지고 있다면 a를 출력
- a || b ==> a,b 중 true인 것을 출력 a=0이라면 0은 false이기때문에 b를 출력

- 함수를 함수의 인수로 전달하고, 필요하다면 인수로 전달한 그 함수를 "나중에 호출(called back)"하는 것이 콜백 함수의 개념입니다.
